# 커밋 및 풀리퀘스트 가이드라인

## 1. 커밋 타입

| 타입       | 설명 |
|------------|------|
| Asset      | 새로운 에셋 추가 (사운드, 스프라이트 등) |
| Feat       | 새로운 기능 추가 (클래스, 메서드 생성 등) |
| Fix        | 버그 수정 (직접적인 오류, 비정상 로직 수정) |
| Refactor   | 기존 클래스 전반적인 리팩토링 및 최적화 |
| Test       | 테스트 코드, 테스트 씬, 테스트 오브젝트 생성 |
| Docs       | 문서 작업 (주석, md 파일 등) |
| Chore      | 위에 해당하지 않는 기타 작업 |

---

## 2. 커밋 메시지 작성 규칙
- **Type**: 위 커밋 타입 참조, 첫 글자 대문자
- **제목**: 50글자 이내로 간결하게 작성, 영어일 경우 첫 글자 대문자
- **본문**: 선택 사항. 제목만으로 설명이 부족할 경우 작성

**작성 가이드**
- “했습니다”,“~~함” 형태 사용X. “기능 작성”, “기능 수정” 등으로 핵심만 전달
- 간결하고 명확하게 작성

Feat : 플레이어의 기능 추가
---

## 3. PR 작성 규칙
### 작성 예시
**개요**: 플레이어 이동 기능 작업/리펙토링/수정  
**변경 사항**
- PlayerMovement.cs 구현 / 수정 / 리펙토링 
- Unity Input System 적용  
- Update 대신 코루틴 사용으로 성능 최적화  
**테스트**
- WASD 키 입력으로 정상 이동 확인  
- 점프와 달리기 기능 정상 작동 확인

### PR 진행
⦁	PR요청
⦁	AI리뷰 혹은 타인의 리뷰 확인
⦁	수정 후 재 확인
⦁	더이상 수정사항이 없을 경우 Merge진행
